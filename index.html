NAV

    Introduction

    This is the documentation of the new Matcha Wine API. It is developed in a RESTful flavor.

    The authentification is done by Basic Authentication, with username and password given by Matcha to the client.

    Wines

    Get Wines

    This endpoint returns JSON structured like this:

    {
  "wines": [
    {
      "id": "34",
      "name": "Bourgogne Rouge Maison Bernard et Fils 2010",
      "color": "R",
      "tags": [
        {
          "name": "Rouge",
          "category": "Color"
        },
        {
          "name": "Léger",
          "category": "Dégustation"
        }
      ]
    },
    {
      "id": "65",
      "name": "Sancerre Blanc Maison Pierre Han 2014",
      "color": "B",
      "tags": [
        {
          "name": "Blanc",
          "category": "Color"
        },
        {
          "name": "Léger",
          "category": "Dégustation"
        }
      ]
    },
  ],
  "remaining_tags": [
    {
      "name": "Rouge",
      "category": "Color"
    },
    {
      "name": "Blanc",
      "category": "Color"
    },
    {
      "name": "Léger",
      "category": "Dégustation"
    }
  ],
  "response_id": "000de456"
}

    This endpoint retrieves all wines if no tags are given. It retrieves all wines that have the given tags otherwise.

    HTTP Request

    GET https://engine.matcha.wine:3030/wines

    Query Parameters

    Parameter Type Description
    tags array[string] Optional. List of tags’ names. If empty or not present, the whole wine set will be returned.
    Success (200)

    Success response attributes

    Parameter Type Description
    wines array[object] Information concerning the wines in the wine set.
    wines[].id string Id of the wine.
    wines[].name string Name of the wine
    wines[].color string Color code of the wine (R, B or Z - for red, white and rosé respectively).
    wines[].tags array[object] Tags of the wine.
    wines[].tags[].name string Name of the tag.
    wines[].tags[].category string Category of the tag.
    remaining_tags array[object] Optional. List containing all tags of the wines sent.
    remaining_tags[].name string Name of the tag.
    remaining_tags[].category string Category of the tag.
    response_id string Unique Id of the request.

    Get a Specific Wine

    This endpoint returns JSON structured like this:

    {
  "wine": {
    "id": "34",
    "name": "Bourgogne Rouge Maison Bernard et Fils 2010",
    "color": "R",
    "tags": [
      {

      }

  },
  "response_id": "000de456"

    This endpoint retrieves all wines if no tags are given. It retrieves all wines that have the given tags otherwise.

    HTTP Request

    GET https://engine.matcha.wine:3030/wines/<wine_id>

    URL Parameters

    Parameter Type Description

    Success Response Attributes

    wine.id string Id of the wine.
    wine.name string Name of the wine
    wine.color string Color code of the wine (R, B or Z - for red, white and rosé respectively).
    wine.tags array[object] Tags of the wine.
    wine.tags[].name string Name of the tag.
    wine.tags[].category string Category of the tag.
    response_id string Unique Id of the request.

    Create a Wine

    This endpoint returns JSON structured like this:

    {
  "wine": {
    "id": "34",
    "name": "Bourgogne Rouge Maison Bernard et Fils 2010",
    "color": "R",
    "tags": [
      {
        "name": "Rouge",
        "category": "Color"
      },
      {
        "name": "Léger",
        "category": "Dégustation"
      }
    ]
  },
  "response_id": "000de456"
}

    This endpoint creates a new wine in the client’s wine set. Wine information like zone, varieties and sugar level are facultative, but they increase the quality of the tags obtained for the wine. If there already is a wine with the given id in the wine set, an error will be sent.

    The input is sent as a JSON. (application/json)

    HTTP Request

    POST https://engine.matcha.wine:3030/wines

    Request Body Json Arguments

    Parameter Type Description
    wine object The wine to be created.
    wine.id string Id of the wine.
    wine.name string Name of the wine.
    wine.color string Color code of the wine (R, B or Z - for red, white and rosé respectively).
    wine.zone string Optional. Geographical zone of the wine - “appellation”.
    wine.varieties string Optional. Grape varieties of the wine.
    wine.sugar string Optional. Sugar level information of the wine.
    wine.sparkling boolean Optional. True if wine is sparkling. Default is “False”.
    Success (200)

    Success Response Attributes

    Parameter Type Description
    wine object The created wine.
    wine.id string Id of the wine.
    wine.name string Name of the wine
    wine.color string Color code of the wine (R, B or Z - for red, white and rosé respectively).
    wine.tags array[object] Tags of the wine.
    wine.tags[].name string Name of the tag.
    wine.tags[].category string Category of the tag.
    response_id string Unique Id of the request.

    Update a Wine

    This endpoint returns JSON structured like this:

    {
  "wine": {
    "id": "34",
    "name": "Bourgogne Rouge Maison Bernard et Fils 2010",
    "color": "R",
    "tags": [
      {
        "name": "Rouge",
        "category": "Color"
      },
      {
        "name": "Léger",
        "category": "Dégustation"
      }
    ]
  },
  "upsert": false,
  "response_id": "000de456"
}

    This endpoint updates a wine in the client’s wine set. The process is the same as in the “Create Wine” endpoint, but the created wine replaces the one with the same id. If there already is no wine with the given id in the wine set, a new wine is created (upsert).

    The input arguments is sent in JSON. (application/json)

    HTTP Request

    PUT https://engine.matcha.wine:3030/wines/<wine_id>

    URL Parameters

    Parameter Type Description
    wine_id string The id of the wine to update.

    Request Body Json Arguments

    Parameter Type Description
    wine object The wine to be updated.
    wine.name string Name of the wine.
    wine.color string Color code of the wine (R, B or Z - for red, white and rosé respectively).
    wine.zone string Optional. Geographical zone of the wine - “appellation”.
    wine.varieties string Optional. Grape varieties of the wine.
    wine.sugar string Optional. Sugar level information of the wine.
    wine.sparkling boolean Optional. True if wine is sparkling. Default is “False”.
    Success (200)

    Success Response Attributes

    Parameter Type Description
    wine object The updated wine (after the update).
    wine.id string Id of the wine.
    wine.name string Name of the wine
    wine.color string Color code of the wine (R, B or Z - for red, white and rosé respectively).
    wine.tags array[object] Tags of the wine.
    wine.tags[].name string Name of the tag.
    wine.tags[].category string Category of the tag.
    upsert boolean “true” if this wine was created.
    response_id string Unique Id of the request.

    response_id string Unique Id of the request.

    Pairings

    Create (and Retrieve) a Wine-Dishes Pairing

    This endpoint returns JSON structured like this:

    {
  "wines": [
      {
      "id": "34",
      "name": "Bourgogne Rouge Maison Bernard et Fils 2010",
      "color": "R",
      "tags": [
        {
          "name": "Rouge",
          "category": "Color"
        },
        {
          "name": "Léger",
          "category": "Dégustation"
        }
      ]
    }
  ],
  "response_id": "000de456"
}

    This endpoint obtains a wine pairing for a dish or for a group of dishes. One or more wines are sent in response, according to wines_nb argument.

    The input arguments is sent in JSON. (application/json)

    HTTP Request

    POST https://engine.matcha.wine:3030/pairings

    Request Body Json Arguments

    Parameter Type Description
    dishes array[string] List of dish descriptions.
    wines_nb integer Optional. Amount of wines to be sent to the match. Default is only one.
    Success (200)

    Success Response Attributes

    Parameter Type Description
    wines array[object] The paired wines, in decreasing order of pairing quality.
    wines[].id string Common id of the wine.
    wines[].name string Name of the wine.
    wines[].color string Color code of the wine (R, B or Z - for red, white and rosé respectively).
    wines[].tags array[object] Tags of the wine.
    wines[].tags[].name string Name of the tag.
    wines[].tags[].category string Category of the tag.
    response_id string Unique Id of the request.

    Errors

    The Matcha API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request’s parameters are not correct.
    401 Unauthorized – Your API key is wrong.
    410 Pairing Invalid Usage – There is no ingredient recognized in at least one of the dishes.
    411 Large Argument Error – There is a single string bigger than 600 characters in the arguments.
    413 Large Request – The request is too large.
    415 Wine Not Found – The wine request does not exist.
    416 Wine Already Exists – Requested creation of a wine that already exists (same wine.id).
    500 Internal Server Error – We had a problem with our server. Try again later.
